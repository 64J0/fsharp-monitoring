# Infrastructure Layer

The **infrastructure layer** adapts external services (PostgreSQL, file system, HTTP clients, …) to the interfaces the application layer expects. It is the outermost layer that knows about implementation details.

## What lives here

| Folder         | Content                                                                             |
| -------------- | ----------------------------------------------------------------------------------- |
| `Generated/`   | F# types auto-generated by **SqlHydra** from the PostgreSQL schema                 |
| `Mappers/`     | Data mapper functions — the **anti-corruption layer** between DB rows and domain    |
| `Repositories/`| Concrete **adapters** implementing the `IXxxRepository` ports from the application  |

## Technology choices

* **SqlHydra** (v4) — generates strongly-typed F# record types from the live database schema, and provides a type-safe CE-based query DSL (`selectTask`, `insertTask`).
* **Npgsql** (v10) — the PostgreSQL .NET driver used by SqlHydra and injected via `NpgsqlDataSource`.
* **QueryContextFactory** — a singleton record type (generated by SqlHydra) that holds the `NpgsqlDataSource` and opens a connection per query.

## Regenerating database types

After applying new migrations, regenerate `Generated/StocksDb.fs` by running:

```bash
cd fsharp-infrastructure
dotnet sqlhydra npgsql
```

or via the Makefile shortcut:

```bash
make db-gen-types
```
