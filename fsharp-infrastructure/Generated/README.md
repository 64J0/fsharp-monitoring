# Generated Types (SqlHydra)

This folder contains **`StocksDb.fs`**, the file auto-generated by **SqlHydra CLI** from the live PostgreSQL schema.

## What it contains

* `CLIMutable` F# record types for each database table (`stocks`, `quotes`, `trades`) — one property per column, using idiomatic F# types (`string`, `decimal`, `DateTimeOffset`, …).
* Table declarations (`let stocks = table<stocks>`) used in SqlHydra's type-safe CE query DSL.
* A `QueryContextFactory` record that wraps `NpgsqlDataSource` and creates a `QueryContext` on demand for each query.

## Why this is NOT hand-written

The generated file is the **only point of contact** between the application and the raw database schema. By regenerating it whenever the schema changes, the compiler enforces that all repositories and mappers stay in sync.

## How to regenerate

1. Ensure PostgreSQL is running with migrations applied (`make db-migrate`).
2. Run from the `fsharp-infrastructure/` directory:

```bash
dotnet sqlhydra npgsql
# or
make db-gen-types   # from the repo root
```

The config for SqlHydra is in `sqlhydra.toml` in `fsharp-infrastructure/`.

> **Do not edit this file manually.** Changes will be overwritten on the next `dotnet sqlhydra npgsql` run.
